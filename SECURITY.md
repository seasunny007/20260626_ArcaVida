# 安全政策 / Politica de seguridad / Security Policy

本文档提供中文、西班牙语和英文说明；如三种文本存在不一致，以中文文本为准。

Este documento ofrece informacion en chino, espanol e ingles. Si existe alguna inconsistencia entre los tres textos, prevalecera el texto chino.

This document provides Chinese, Spanish, and English information. If there is any inconsistency among the three texts, the Chinese text controls.

ArcaVida 处于社区预览阶段。请不要在公开 Issue、PR、讨论区或聊天中提交真实凭证、真实求助消息、电话号码、精确位置、身份证件、服务账号 JSON、API key、bot token、本地数据库或日志。

ArcaVida esta en vista previa comunitaria. No envies credenciales reales, mensajes reales de solicitud de ayuda, telefonos, ubicaciones exactas, documentos de identidad, archivos JSON de cuentas de servicio, API keys, bot tokens, bases de datos locales ni logs en issues, pull requests, discusiones o chats publicos.

ArcaVida is in community preview. Do not submit real credentials, real help-request messages, phone numbers, exact locations, identity documents, service account JSON files, API keys, bot tokens, local databases, or logs in public issues, pull requests, discussions, or chats.

## 报告安全问题 / Reportar un problema de seguridad / Reporting a Security Issue

如果你发现安全问题，请优先使用 GitHub 的私密漏洞报告或 Security Advisories 流程。不要在公开 Issue 中披露可利用细节、真实个人数据或真实凭证。

Si encuentras un problema de seguridad, usa preferentemente el reporte privado de vulnerabilidades de GitHub o Security Advisories. No reveles detalles explotables, datos personales reales ni credenciales reales en issues publicos.

If you find a security issue, prefer GitHub private vulnerability reporting or Security Advisories. Do not disclose exploitable details, real personal data, or real credentials in public issues.

## 范围 / Alcance / Scope

可报告的问题包括 / Los problemas reportables incluyen / Reportable issues include:

- 凭证、token、密钥或本地路径泄露风险。 / Riesgos de exposicion de credenciales, tokens, secretos o rutas locales. / Credential, token, secret, or local-path exposure risks.
- 未授权访问、会话或部署配置问题。 / Problemas de acceso no autorizado, sesion o configuracion de despliegue. / Unauthorized access, session, or deployment-configuration issues.
- 可能导致个人数据泄露或重新识别的缺陷。 / Defectos que puedan exponer o reidentificar datos personales. / Defects that may expose or re-identify personal data.
- 日志、数据库、部署配置或导出流程中的隐私风险。 / Riesgos de privacidad en logs, bases de datos, configuracion de despliegue o flujos de exportacion. / Privacy risks in logs, databases, deployment configuration, or export flows.

## 公开讨论边界 / Limite de discusion publica / Public Discussion Boundary

公开讨论只应使用合成、公开或充分匿名化样例。ArcaVida 不是公众求助入口、现场救援系统、救援派遣渠道或真实个案处理平台。

La discusion publica debe usar solo ejemplos sinteticos, publicos o suficientemente anonimizados. ArcaVida no es un portal publico de solicitud de ayuda, sistema de rescate en campo, canal de despacho ni plataforma de gestion de casos reales.

Public discussion should use only synthetic, public, or fully anonymized examples. ArcaVida is not a public help-request portal, field rescue system, dispatch channel, or real-case handling platform.
